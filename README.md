# Natural2SQL: RL-AIF Training Sandbox

Natural2SQL is a **multi-turn reinforcement learning environment** designed to train LLM agents in professional SQL generation, schema reasoning, and iterative self-correction.

Unlike static benchmarks, this system functions as an **interactive sandbox**, where agents learn through trial, compiler feedback, and reward shaping.

---

## Core Training Philosophy

The environment is built around **Iterative Self-Correction** and **Reinforcement Learning from AI Feedback (RL-AIF)**.

Agents are evaluated not only on correctness, but also on:
- How they navigate schema complexity  
- How they respond to execution feedback  
- How efficiently they construct queries  

---

## Innovation Highlights

### Safety Guardrails
LLM agents would often write lazy or dangerous SQL commands. The penalizing guardrails solve that. We have demonstrated that this extends to production safety, not just code golfing.

- Strict penalty (-1.0) for destructive SQL operations:

```js
blocked_patterns = [
    "DROP", "DELETE", "TRUNCATE", "UPDATE", "INSERT",  # Destructive
    "UNION",                                           # Unauthorized Extraction/Leakage
    "CROSS JOIN",                                      # Cartesian Product (Resource Exhaustion)
    "ALTER", "GRANT", "REVOKE", "CREATE"               # Schema/Permission manipulation
]
```
- Simulates production-safe SQL environments

### Efficiency Grading
Grades efficient and optimized SQL queries.

- Penalizes `SELECT *`
- Rewards explicit column selection
- Encourages production-grade query writing

### Compiler-in-the-Loop Feedback
- SQLite execution errors are returned to the agent
- Enables multi-turn debugging and correction
- Mimics real-world development workflows

## Environment Specification

### Observation Space
A structured Pydantic-based state containing:
- `prompt`: Natural language SQL task
- `last_execution_result`: Output or compiler feedback
- `remaining_attempts`: Number of retries left

### Action Space
- A single SQL query string generated by the agent

### Reward Function and hierarchy

| Condition | Reward | Description |
|----------|--------|------------|
| Success (optimized) | 1.0 | Correct data + efficient query |
| Success (inefficient) | 0.8 | Correct but uses `SELECT *` |
| Misalignment | 0.3 | Valid SQL syntax but incorrect logic or schema usage |
| Syntax Error | -0.3 | Query fails to compile; triggers retry |
| Safety Violation / Failure | -1.0 | Destructive query or exhausted attempts |

## Task Curriculum

### Level 1 — Easy
- Single-table filtering
- Example: Customer ID lookup

### Level 2 — Medium
- Aggregations using `SUM`, `AVG`, `GROUP BY`
- Example: Total revenue computation

### Level 3 — Hard
- Multi-table joins with constraints
- Includes `DISTINCT` usage

### Level 4 — Super Hard
- Multi-hop relational reasoning
- Example: Customer → Orders → Items traversal
